Knowledge:
- 다음 텍스트 데이터를 Cold Rolled 제품 유형이 요구하는 구조화된 라인 아이템 JSON 형식으로 변환합니다.
- 설명 텍스트나 코드 포맷 없이 원시 JSON만 제공합니다.
- Cold Rolled 유형은 14개의 논리 데이터(필드)를 가지고 있습니다.
  - 품종, INQ규격명, 주문용도, 두께, 폭, 수량, 주문(Edge), 내경, 외경, 슬리브두께, 희망납기일, 인장강도, 연신율, 경도
- 그리고 위 14개의 필드는 물리 데이터와 매핑됩니다.
  - kind, inqName, orderCategory, thickness, width, quantity, expectedDeadline, orderEdge, inDiameter, outDiameter, sleeveThickness, tensileStrength, elongationRatio, hardness
- 14개의 데이터가 모여 하나의 라인 아이템이 완성됩니다.
- 추출된 텍스트 데이터에는 여러 개의 라인 아이템이 존재합니다.
- 여러 개의 라인 아이템을 만들 수 있는 경우 각각을 `lineItemResponseDTOs` 배열에서 별도의 개체로 표현해야 합니다.
- 라인 아이템이 하나일 경우, 예시 응답 구조는 아래와 같습니다.
  - {
      "lineItemResponseDTOs": [
        {
          "kind": "CR",
          "inqName": "JS_SI123",
          "orderCategory": "파이프 소재",
          "thickness": "1.5mm",
          "width": "1200mm",
          "quantity": 500,
          "expectedDeadline": "2024-09-15",
          "orderEdge": "Mill Edge",
          "inDiameter": "500mm",
          "outDiameter": "600mm",
          "sleeveThickness": "2mm",
          "tensileStrength": "240Mpa",
          "elongationRatio": "15%",
          "hardness": "180HV"
        }
      ]
    }

LineItem Field Detail Information:
- 품종(논리) = kind(물리) 필드는 'CR', 'CRC', 'CRCA' enum값을 가지고 있다.
- INQ규격명(논리) = inqName(물리) 필드는 'JS_SI123', 'JS_SI456', 'JS_SI789', 'JS_SI321', 'JS_SI654', 'JS_SI987' enum값을 가지고 있다.
  - 만약, INQ규격명의 enum값 내 공백이 존재하는 경우(JS SI123) '_'로 붙여 요구하는 enum값 형식을 맞춰줘야 한다.
  - INQ규격명은 'JS_SIxxx' 포맷을 가지고 있으며, xxx는 정수형이다.
- 주문용도(논리) = orderCategory(물리) 필드는 '자동차 부품' 또는 '건축용 외장재'과 같은 냉연(Cold Rolled) 제품에 대한 사용 목적을 의미하는 값을 가져야 한다.
- 두께(논리) = thickness(물리) 필드는 15mm와 같이 냉연 부품의 두께를 의미하며, 'mm' 단위를 포함한 값을 가져야 한다.
  - 만약, 'mm' 단위를 포함하지 않은 경우 단위를 포함해 반환한다.
- 폭(논리) = width(물리) 필드는 1800mm와 같이 냉연 부품의 폭을 의미하며, 'mm' 단위를 포함한 값을 가져야 한다.
  - 만약, 'mm' 단위를 포함하지 않은 경우 단위를 포함해 반환한다.
- 수량(논리) = quantity(물리) 필드는 200과 같이 자동차 부품 수량을 의미하며, 정수형 값을 가져야 한다.
- 희망납기일(논리) = expectedDeliveryDate(물리) 필드는 날짜형 데이터로, yyyy-MM-dd 형태의 값을 가져야 한다.
- 주문(논리) = orderEdge(물리) 필드는 자동차 부품이 요구하는 주문 형식으로, Edge가 포함된 값을 가져야 한다.
- 내경(논리) = inDiameter(물리) 필드는 냉연 두께의 안쪽 지름을 의미하며, 'mm' 단위를 포함한 값을 가져야 한다.
  - 만약, 'mm' 단위를 포함하지 않은 경우 단위를 포함해 반환한다.
- 외경(논리) = outDiameter(물리) 필드는 냉연 두께의 바깥쪽 지름을 의미하며, 'mm' 단위를 포함한 값을 가져야 한다.
  - 만약, 'mm' 단위를 포함하지 않은 경우 단위를 포함해 반환한다.
- 슬리브두께(논리) = sleeveThickness(물리) 필드는 배관이 관통할 수 있는 관을 의미하며, 'mm' 단위를 포함한 값을 가져야 한다.
  - 만약, 'mm' 단위를 포함하지 않은 경우 단위를 포함해 반환한다.
- 인장강도(논리) = tensileStrength(물리) 필드는 재료가 파괴되기 전 버틸 수 있는 최대 인장 응력을 의미하며, 'Mpa' 단위를 포함한 값을 가져야 한다.
  - 만약, 'Mpa' 단위를 포함하지 않은 경우 단위를 포함해 반환한다.
- 연신율(논리) = elongationRatio(물리) 필드는 인장시험 후 시료가 얼마나 늘어났는지를 판단하는 값을 의미하며, '%' 단위를 포함한 값을 가져야 한다.
  - 만약, '%' 단위를 포함하지 않은 경우 단위를 포함해 반환한다.
- 경도(논리) = hardness(물리) 필드는 힘을 가했을 때 저항하는 힘의 정도를 의미하며, 'HV' 단위를 포함한 값을 가져야 한다.
  - 만약, 'HV' 단위를 포함하지 않은 경우 단위를 포함해 반환한다.


- 항상 참조할 수 있는 사실적 진술을 말합니다.
- 들어온 데이터를 활용해 하나의 라인 아이템을 만들 수 있다면, 모든 조합을 통해 빠짐없이 라인아이템을 생성해야 합니다.
- 임의로 데이터를 만들거나 수정하면 안 됩니다.
- 순서 구분 없이 데이터가 들어오더라도, 위 Cold Rolled 유형의 라인 아이템이 가져야 하는 데이터 형식에 맞게 변환되어야 합니다.
- 들어온 데이터는 빠짐없이 응답 구조에 들어가야 하며, 여러 개의 라인 아이템이 만들어지는 경우 `lineItemResponseDTOs` 배열에서 별도의 개체로 표현해야 합니다.
- 중복된 라인 아이템이 만들어질 수 있어도 각 데이터를 개별적인 라인 아이템으로 인식해야 한다.
  - 그리고, 중복된 라인 아이템의 경우 `lineItemResponseDTOs` 배열에서 별도의 개체로 표현해야 한다.
