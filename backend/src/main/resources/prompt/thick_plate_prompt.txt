Knowledge:
- 다음 텍스트 데이터를 'Thick Plate' 제품 유형형 요구하는 구조화된 라인 아이템 JSON 형식으로 변환합니다.
- 설명 텍스트나 코드 포맷 없이 원시 JSON만 제공합니다.
- 'Thick Plate' 유형은 14개의 논리 데이터(필드)를 가지고 있습니다.
  - 일반사항, 주문정보, 성분(ladle), 성분(product), 인장, 입도시험, 충격, 추가충격, 시효충격, 굴곡, 추가요청사항, 경도, 낙하충격시험, 초음파탐상시험
- 그리고 위 14개의 필드는 물리 데이터와 매핑됩니다.
  - orderPurpose, orderInfo, ladleIngredient, productIngredient, seal, grainSizeAnalysis, show, extraShow, agingShow, curve, additionalRequests, hardness, dropWeightTest, ultrasonicTransducer
- 14개의 데이터가 모여 하나의 라인 아이템이 완성됩니다.
- 추출된 텍스트 데이터에는 여러 개의 라인 아이템이 존재합니다.
- 여러 개의 라인 아이템을 만들 수 있는 경우 각각을 `lineItemResponseDTOs` 배열에서 별도의 개체로 표현해야 합니다.
- 라인 아이템이 하나일 경우, 예시 응답 구조는 아래와 같습니다.
  - {
      "lineItemResponseDTOs": [
        {
          "orderPurpose": "교량용",
          "orderInfo": "TP001",
          "ladleIngredient": "마그네시아",
          "productIngredient": "Carbon",
          "seal": "400Mpa ~ 450Mpa",
          "grainSizeAnalysis": true,
          "show": "27 J @ -20°C",
          "extraShow": "40 J @ -30°C",
          "agingShow": "35 J @ -40°C",
          "curve": "500Mpa",
          "additionalRequests": "최대한 빠른 납부 바람",
          "hardness": "200HV",
          "dropWeightTest": true,
          "ultrasonicTransducer": true
        }
      ]
    }

LineItem Field Detail Information:
- 일반사항(논리) = orderPurpose(물리) 필드는 '교량용', '선박용도'와 같은 값을 가지고 있다.
- 주문정보(논리) = orderInfo(물리) 필드는 'TP001', 'TP002' enum값을 가지고 있다.
- 성분(ladle)(논리) = ladleIngredient(물리) 필드는 '마그네시아' 또는 '알루미나'과 같은 후판(Thick Plate) 제품에 대한 성분을 의미하는 값을 가져야 한다.
- 성분(product)(논리) = productIngredient(물리) 필드는 'Carbon', 'Slicon'과 같은 후판(Thick Plate) 제품에 대한 성분을 의미하는 영문 값을 가져야 한다.
- 인장(논리) = seal(물리) 필드는 후판의 끌어당김 정도를 의미하며, 'X MPa ~ Y MPa'단위를 포함한 값을 가져야 한다.
  - 만약, 'MPa' 단위를 포함하지 않은 경우 단위를 포함해 반환한다.
- 입도시험(논리) = grainSizeAnalysis(물리) 필드는 Boolean 타입의 값을 가져야 한다.
- 충격, 추가충격, 시효충격 구분
  - 충격(논리) = show(물리) 필드는 물체에 급격하게 가해지는 정도를 의미하며, 데이터가 's'로 시작하는 경우 'show' 필드에 입력해야 한다.
  - 추가충격(논리) = extraShow(물리) 필드는 물체에 급격하게 가해지는 추가 충격을 의미하며, 데이터가 'es'로 시작하는 경우 'extraShow' 필드에 입력해야 한다.
  - 시효충격(논리) = agingShow(물리) 필드는 물체에 급격하게 가해지는 시효 충격을 의미하며, 데이터가 'as'로 시작하는 경우 'agingShow' 필드에 입력해야 한다.
  - 위 3가지 필드는 해당 필드에 데이터를 입력하기 전에 's', 'es' 또는 'as' 접두사를 제거해야한다.
- 굴곡(논리) = curve(물리) 필드는 제품의 구부러진 정도를 의미하며, 'Mpa' 단위를 포함한 값을 가져야 한다.
  - 만약, 'Mpa' 단위를 포함하지 않은 경우 단위를 포함해 반환한다.
- 추가요청사항(논리) = additionalRequests(물리) 필드는 제품 구매에 대한 요청 사항을 의미하며, 하나의 요청만을 적을 수 있다.
- 경도(논리) = hardness(물리) 필드는 물질의 굳기 정도를 의미하며, 'HV' 단위를 포함한 값을 가져야 한다.
  - 만약, 'HV' 단위를 포함하지 않은 경우 단위를 포함해 반환한다.
- 낙하충격시험(논리) = dropWeightTest(물리) 필드는 Boolean 타입의 값을 가져야 한다.
- 초음파탐상시험(논리) = ultrasonicTransducer(물리) 필드는 Boolean 타입의 값을 가져야 한다.


- 항상 참조할 수 있는 사실적 진술을 말합니다.
- 들어온 데이터를 활용해 하나의 라인 아이템을 만들 수 있다면, 모든 조합을 통해 빠짐없이 라인 아이템을 생성해야 합니다.
- 임의로 데이터를 만들거나 수정하면 안됩니다.
- 순서 구분 없이 데이터가 들어오더라도, 위 Thick Plate 유형의 라인 아이템이 가져야 하는 데이터 형식에 맞게 변환되어야 합니다.
- 들어온 데이터는 빠짐없이 응답 구조에 들어가야 하며, 여러 개의 라인 아이템이 만들어지는 경우 `lineItemResponseDTOs` 배열에서 별도의 개체로 표현해야 합니다.
- 중복된 라인 아이템이 만들어질 수 있어도 각 데이터를 개별적인 라인 아이템으로 인식해야 한다.
  - 그리고, 중복된 라인 아이템의 경우 `lineItemResponseDTOs` 배열에서 별도의 개체로 표현해야 한다.
